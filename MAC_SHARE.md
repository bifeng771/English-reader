# Mac 用户打开说明

不要直接点 `http://127.0.0.1:4179/`。这个地址只有在 Mac 已经启动本地服务后才会打开。

## 推荐步骤：不用拖文件夹

1. 解压压缩包。
2. 在 Finder 里找到解压后的文件夹，也就是能看到 `index.html`、`app.js`、`vendor` 的那个文件夹。
3. 选中这个文件夹。
4. 按住 `Option` 键，同时右键文件夹，选择“复制 ... 作为路径名”。也可以直接按 `Option + Command + C`。
5. 打开 Mac 的“终端”。
6. 输入 `cd `，注意后面有一个空格，然后粘贴刚才复制的路径，按回车。
7. 输入：

```bash
python3 -m http.server 4179
```

8. 保持终端不要关闭。
9. 再用 Safari 或 Chrome 打开：

```text
http://127.0.0.1:4179/
```

## 启动脚本方式

在终端进入解压后的文件夹后，也可以运行：

```bash
zsh start-mac.command
```

这个脚本会自动打开浏览器，并启动本地服务。

## Finder 右键打开终端

如果右键文件夹后能看到“服务”或“快速操作”，可以选择“新建位于文件夹位置的终端窗口”。终端打开后直接运行：

```bash
python3 -m http.server 4179
```

## 为什么压缩包里的网址不能直接打开

因为压缩包只是网页文件，不是云端网站。`127.0.0.1` 代表“当前电脑自己”，没有启动服务时它没有内容可访问。

想让别人双击链接就能打开，需要把整个文件夹部署到公网 HTTPS，例如 Netlify、Vercel、Cloudflare Pages 或 GitHub Pages。
