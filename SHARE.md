# 分享和打开说明

请先看这一句：`http://127.0.0.1:4179/` 不是公网网址，不能直接发给别人打开。

`127.0.0.1` 的意思是“当前这台电脑自己”。所以苹果电脑用户收到压缩包后，必须先在自己的 Mac 上启动本地网页服务，然后再打开这个网址。

## Mac 用户本地打开

方法一：复制路径，不需要拖文件夹。

1. 解压压缩包。
2. 在 Finder 里找到解压后的文件夹，也就是能看到 `index.html`、`app.js`、`vendor` 的那个文件夹。
3. 选中这个文件夹。
4. 按住 `Option` 键，同时右键文件夹，选择“复制 ... 作为路径名”。也可以直接按 `Option + Command + C`。
5. 打开 Mac 的“终端”。
6. 输入 `cd `，注意 cd 后面有一个空格，然后粘贴刚才复制的路径，按回车。
7. 输入下面这行并按回车：

```bash
python3 -m http.server 4179
```

8. 不要关闭终端窗口。
9. 在 Safari 或 Chrome 打开：

```text
http://127.0.0.1:4179/
```

方法二：用启动脚本。

1. 解压压缩包。
2. 打开终端。
3. 按上面的方法复制文件夹路径，输入 `cd ` 后粘贴路径，按回车。
4. 运行：

```bash
zsh start-mac.command
```

这个方式不需要先执行 `chmod`。之后也可以尝试双击 `start-mac.command` 启动。若 Mac 提示安全限制，请在终端用上面的命令启动。

方法三：Finder 右键打开终端。

1. 右键解压后的文件夹。
2. 选择“服务”或“快速操作”。
3. 选择“新建位于文件夹位置的终端窗口”。
4. 终端打开后运行：

```bash
python3 -m http.server 4179
```

## Windows 用户本地打开

在文件夹里运行：

```powershell
.\run-local.ps1
```

然后打开：

```text
http://127.0.0.1:4179/
```

## 想发给别人直接打开

需要把整个文件夹部署到公网 HTTPS，例如 Netlify、Vercel、Cloudflare Pages 或 GitHub Pages。部署成功后会得到一个类似下面的公网地址：

```text
https://你的站点.netlify.app/
```

这个公网地址才能直接发给苹果电脑、Windows 电脑和手机用户打开。

## 常见错误

- 直接点 `http://127.0.0.1:4179/` 打不开：还没有启动本地网页服务。
- 终端一关闭网页就打不开：本地服务已经停止。
- 发给别人 `127.0.0.1` 地址打不开：这个地址只指向对方自己的电脑，不会指向你的电脑。
- 想让别人不用终端：需要部署到公网 HTTPS。
